Data pipeline architecture

Google Ads & Meta Ads → SCD2 warehouse

One Step Functions pipeline, platform kept as a fan-out dimension rather than duplicated per ad network. Traces every record from the daily schedule trigger through validation to the warehouse and the dashboard that reads it.

One pipeline, not two

A single Map state fans out over {platform, account}. The IsAsync choice is the only point where Google Ads' synchronous GAQL query and Meta Ads' async create/poll/download job actually diverge.

SCD2 history is earned, not faked

The demo window loads in two separate simulated passes split at a campaign rename, so dim_campaign's version chain reflects two real loads, not a hand-written fixture.

Nothing calls a real ad API

DEMO_MODE=1 stubs credential resolution. No AWS resources are provisioned — the warehouse step runs against SQLite as a local stand-in for Redshift Serverless.
